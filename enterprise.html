Where It Fits

Use TrueLock when the issue is control after sending

After a file is sent through email, messenger, cloud storage, or a shared drive, the sender usually loses control. TrueLock is for workflows where content should stay locked until the right moment or context.

Event and promo drops

Files open only at the venue or only in a defined time window.

Partner and contractor handoffs

Delay access until a meeting, deadline, or physical location.

Internal release windows

Release packages stay locked until the intended internal launch window.

Sensitive client files

Recipient-controlled, offline-first delivery instead of cloud-first exposure.

Pilot Proposal

Start with one workflow, not a platform rollout

The fastest evaluation path is a focused pilot around one business scenario. This keeps scope honest and makes the result easy to judge.

1 use case

Choose one operational pain point: event drop, delayed handoff, release window, or sensitive file delivery.

1 workflow

Define how the capsule is created, distributed, and opened in practice.

1 stakeholder group

Limit the pilot to one team or one recipient group to avoid fuzzy feedback.

1 short validation period

Run a short test with concrete success criteria instead of a long enterprise process.

Product Facts

Procurement-ready summary

Platforms

Current public release scope: Android and Windows.

Core rules

Password, time window, geo-based access, and combined AND/OR logic.

Recipient model

Recipients can open and verify capsules for free. No mandatory account is required for basic opening.

Cloud dependency

Offline-first delivery model. No mandatory TrueLock cloud is required for core capsule delivery.

Trust boundary

Access enforcement is application-layer. A fully compromised client or runtime can bypass local controls.

Security references

Public format, policy, security, verification, and trust documents are available on the main site.

B2B FAQ

Questions procurement teams and AI agents ask first

Does TrueLock publish enterprise pricing?

Not as a flat public number yet. Free and Pro are public. Business pilots, white-label work, and licensing are scoped individually because the commercial shape depends on workflow and delivery model.

Is there a team or business entry point?

Yes. The recommended path is a focused pilot around one workflow. That keeps the evaluation short and the result measurable.

Is there an SDK or API?

There is no public self-serve API published on the site today. White-label, format, and integration discussions are handled directly.

Does TrueLock require cloud storage of sensitive files?

No. The core model is offline-first capsule delivery through existing channels such as email, messenger, drive links, or physical transfer.

What should a buyer review before trusting it?

Start with the pricing resources on this page, then review the public capsule format, policy rules, security model, release verification, and trust pack documents.
